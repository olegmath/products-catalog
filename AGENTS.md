# Проект Олега

## Self-improvement protocol
- После любой моей коррекции ты обязан предложить лаконичное правило и дописать его в этот AGENTS.md в подходящую секцию.
- Формат правила: одно императивное предложение, без обоснования и без примеров, если случай не двусмысленный.
- Перед добавлением правила выполни поиск по этому файлу. Если уже есть правило, покрывающее этот случай, не дублируй, а уточни существующее.
- Держи общий объём AGENTS.md в пределах 2500 токенов. При превышении объедини пересекающиеся правила и удали устаревшие.
- После фикса бага записывай правило про класс корневой причины, а не про конкретный симптом.

## API
- Проверяй актуальную доступность продукта в API перед выводом, что продукт отсутствует.
- Получай идентификаторы клиентов из API по данным формы вместо использования жёстко заданных clientId.
- Создавай заказ на клиента-получателя обучения, когда покупатель и ученик отличаются.
- Передавай покупателя как customerId, а получателя обучения как clientId при создании заказа.
- Используй существующий объект из ответа API при повторном создании клиента вместо остановки сценария.
- Показывай мультивыбор заказа только через API-контракт, который реально создаёт все выбранные позиции.
- Передавай обязательные fieldValues каталожной позиции при создании заказа через master GraphQL.
- Используй платежную ссылку из созданного заказа перед созданием новой платежной транзакции.

## UI
- Удаляй неиспользуемые элементы управления из интерфейса вместо того, чтобы оставлять их неактивными.
- Отделяй учебные занятия от экзаменационных дней при переносе расписания в интерфейс.
- Не изменяй даты, время и количество занятий при обновлении поясняющих текстов карточек.
- Переводи пользователя на внешний платежный URL сразу после успешного создания платежа.
